# 팀보드 시작하기 — 처음 30분

플랫폼사업팀 업무 관리 도구입니다. 이 문서만 읽으면 오늘 바로 쓸 수 있습니다.

## 1. 로그인 (2분)

1. 팀장에게 받은 **업무메일 + 임시 비밀번호**로 로그인합니다.
2. 첫 로그인이면 **비밀번호 변경 화면**이 먼저 뜹니다. 새 비밀번호(8자 이상)로 바꿔야 계속 진행됩니다.
3. 변경하면 바로 홈으로 들어갑니다.

## 2. 화면 5개만 알면 됩니다

| 화면 | 여기서 하는 일 |
|---|---|
| **홈** | 팀 전체 현황 한눈에 — 오늘 일정, 각자 업무, 마감 임박, 시그널. 하루를 여기서 시작하세요. |
| **캘린더** | 일정(시간 있는 것)과 업무(기한만 있는 것)를 2층 레인으로 봅니다. 일·주·월 전환. |
| **내 업무** | 내가 할 일 목록. 상태(대기→진행→리뷰→완료)를 바꾸고, 목표에 연결합니다. |
| **내 부사수** | 자료조사·회의록·정리 같은 부수 업무를 AI에게 맡깁니다. **초안이 나오면 내가 승인해야** 팀 기록(Notion)에 올라갑니다. |
| **허들** | 아직 정해지지 않은 것을 팀에 던지고 의견을 받습니다. 코멘트가 그대로 결정 근거로 이어집니다. |

> 목표·월간 보고·구성원 관리는 팀장 전용이라 지금은 몰라도 됩니다.

## 3. "이걸 어디에 적지?" — 3초 판단

- **할 일이 정해졌다** → **내 업무**(Task)에 적는다.
- **정해지지 않았고 의견이 필요하다** → **허들**에 올린다.
- **나만 기억하면 된다** → **내 메모**(시그널에서 메모로 작성, 나에게만 보임).

메모는 나중에 "허들로 보내기 → 결정으로 승격 → 업무 생성"으로 자연스럽게 이어집니다. 처음부터 완벽히 분류하지 않아도 됩니다.

## 4. 자주 하는 실수

1. **부사수 초안을 승인 안 하고 기다림** — 부사수는 초안만 만듭니다. 내가 **승인**을 눌러야 팀 기록에 반영됩니다. "보고 대기"에 쌓여 있으면 확인하세요.
2. **정해지지 않은 걸 업무로 만듦** — "이거 어떻게 할까?" 단계는 업무가 아니라 **허들**입니다. 업무는 담당·기한이 정해진 실행 단위입니다.
3. **업무를 목표에 안 엮음** — 월 목표에 업무를 연결하면 진척이 자동 집계됩니다(선택 사항이지만 하면 팀장이 따로 묻지 않습니다).

## 오늘 할 일

① 비밀번호 바꾸기 → ② 홈에서 내 레인 확인 → ③ 오늘 할 일 1개를 **내 업무**에 적기 → ④ 부사수에게 부수 업무 하나 맡겨보기. 끝.
